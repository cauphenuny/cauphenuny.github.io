Review

T1 难度较低，写了正解，T2、T3写了暴力，不知道 T2 为什么没有分，120pts.
要注意看题，差点就把 T1 看错了，导致误判难度。还有要注意题目中的数据范围，比如 T2 k≤5k\le5k≤5 就是一个很好的入手条件。

Solution

T1 god

在平面直角坐标系中，在 x 负半轴有一些位置，编号 111 ~ nnn ，正半轴有 mmm 个点 (si,0)(s_i,0)(si​,0) 。每个点有一个移动向量 (xi,yi)(x_i,y_i)(xi​,yi​) 每一秒按此向量移动 (0≤xi,yi≤1050\le x_i,y_i\le 10^50≤xi​,yi​≤105) ，每个点的价值为它到编号为 [li,ri][l_i,r_i][li​,ri​] 区间内位置的曼哈顿距离和，qqq 个询问，每个询问给定 ttt ，求 yyy 时刻每个点价值的最小值。

设点 i 对时间的价值函数为 valival_ivali​

由于点只会往右上方跑，所以 valival_ivali​ 是一个一次函数，对所有的一次函数按斜率由大到小排序，用栈维护上凸壳即可。

查询时二分位置，找到当前最小的函数 valkval_kvalk​ ，则答案为 valk(t)val_k(t)valk​(t) 。

overwatch

给一颗 nnn 个点的树，点有权值 aia_iai​，定义一条链的值为链上所有点权值和。

qqq 个询问，每个询问两个数 w,kw,kw,k。改变一个点的权值为 www ，求每次改变后的树中长度为 kkk (点的个数为 kkk) 的链的最大值。

n,q≤105n,q\le10^5n,q≤105，k≤5k\le5k≤5

kkk 的大小不超过 555 ，这是一个很好的入手条件。

发现对于点 xxx 的每个儿子,里面可能有很多长度为 kkk 的小路径可以接上点 xxx 变成一条长度为 k+1k + 1k+1 的小路径,但是我们只需要那条权值最大的就行了。所以对于每个点,我们开 kkk 个 set (每个长度开一个),每个 set 的大小为这个点的孩子个数,把每个儿子当前长度权值最大的那条小路径塞进去,然后计算答案的话拼一拼就好了 (还是要特判来自同一子树)。
对于一次修改 (x,w)(x, w)(x,w), 只会影响到点 xxx 以及 Ta 的 1…(k−1)1\ldots(k − 1)1…(k−1) 级祖先,我们先把原来最长的小路径删去,然后再把现在新的最长的小路径加上就好了,对于每个被修改的点,以 Ta 为 lca 的大路径的答案可能发生变化,所以我们可以在外层写一个可删堆,里面塞上以每个点为 lca 的大路径的答案。那么现在对于每个被修改的点,只要把原先在外层可删堆里的答案删掉,再重新拼一拼计算出新的答案,塞回外层可删堆里就好了。
